## 数据状态说明

- `official`：已记录官方来源的数据或待复核的官方来源线索。
- `generated`：基于 official 数据生成的产品模型数据，不属于官方数据。
- `changelog`：每次参数维护记录。

生成数据补齐任务清单：

```bash
npm run data:backfill-tasks
```

校验清单是否与当前源数据一致：

```bash
npm run verify:data-backfill-tasks
```

当前数据任务以 `data/generated/city-data-backfill-tasks.json` 为准。上海公开支持范围内任务为发布前 P0；2000-2007、2013 等支持范围外早年历史参数为 P1 backlog，触达这些年份的输入由小程序提交前闸门拦截。
部分 `fill-missing-fields` 任务包含 `sourceLeads`，仅作为下一轮复核的链接和搜索线索；字段不能因为存在线索就移出 `missingFields`，必须在 `sources.json` 记录可复核来源并更新参数后再放行。

当前主要状态：

- `pending_second_review`：已采集，待二次复核。
- `pending_city_applicability_review`：来源已采集，但还需确认城市适用性。
- `partial`：该年份仅部分字段已维护。
- `missing_pending_backfill`：字段缺失，待回补。

## 重要原则

用户输入可以粗略，但系统基础数据不能偷懒。历史年度参数缺失时，必须显式标记，不允许静默使用最新年份参数替代全部历史。
