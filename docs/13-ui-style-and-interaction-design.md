# 13 UI 风格与交互设计

更新时间：2026-05-28

本文档记录“退休金算盘 / Pension Abacus”微信小程序 V1 的视觉风格、交互原则和页面布局方案。

## 一、设计原则

V1 设计目标不是做一个冷冰冰的表单，也不是做成理财营销或游戏排行产品，而是做一个可信、清楚、低压力的养老金快速测算工具。

核心原则：

- 可信优先：结果页优先展示估算金额、估算范围、拆分项、关键假设、数据质量和免责声明。
- 能做尽做：在不误导用户、不突破数据门槛的前提下，尽量增强仪式感、分享性和可理解性。
- 低门槛：用户只需填写最必要的信息，不知道也可以继续。
- 适老友好：字号足够大、对比清楚、触控区域宽松、提示温和。
- 有边界：所有未纳入精算的项目必须明确提示，不能伪装成官方核定结果。

## 二、推荐视觉方向

推荐方向：政务可信 + 轻算盘品牌感。

气质关键词：

- 可信
- 克制
- 清楚
- 稍有仪式感
- 不像保险营销
- 不像游戏抽奖

不采用：

- 大面积金色理财风。
- 全屏烟花、彩带、奖杯的游戏化风格。
- 过度柔和的家庭关怀插画风。

## 三、视觉规范

### 色彩

| 用途 | 色值 | 说明 |
| --- | --- | --- |
| 主色 | `#176B73` | 深青蓝，用于主按钮、重点状态、品牌识别 |
| 强调深蓝 | `#17324D` | 标题、金额、重要文字 |
| 背景 | `#F6F8FA` | 页面背景 |
| 卡片 | `#FFFFFF` | 内容容器 |
| 边线 | `#D8E1E8` | 分割线、输入框边界 |
| 提醒金 | `#B7791F` | 轻微仪式感、算盘符号、提示重点 |
| 风险红 | `#B42318` | 风险提示、不可用状态 |
| 成功绿 | `#1F7A4D` | 数据已复核、可测算状态 |

暖金只用于小面积强调，例如算盘符号、竞争力徽章、轻量奖杯，不做大面积背景。

### 字体层级

- 页面标题：40-44rpx。
- 模块标题：34-36rpx。
- 正文：30-32rpx。
- 辅助说明：不低于 26-28rpx。
- 结果金额：64-72rpx。
- “元/月”：30-34rpx。

### 间距与触控

- 页面左右边距：32rpx。
- 区块间距：40rpx。
- 表单项高度：104-120rpx。
- 主按钮高度：96rpx。
- 最小触控热区：88rpx。
- 底部固定按钮必须避开小程序安全区。
- 表单错误提示显示在对应字段下方，不只用 toast。
- 长表单和结果页必须支持自然滚动，底部按钮不遮挡最后一项内容。

避免卡片套卡片。结果页主信息可以使用整块浅色区域承载，不要堆叠多个浮层。

### 圆角

- 卡片：12-16rpx。
- 按钮：12-16rpx。
- 输入框：12rpx。

不要使用过度圆润的糖果风。

### 图标

使用线性图标为主：

- 计算
- 日历
- 城市
- 账户
- 数据来源
- 提示
- 分享
- 奖杯 / 徽章

算盘可以作为品牌小符号，不做复杂吉祥物。

### 动效

- 普通交互使用 150-220ms 淡入、上滑、数字轻微递增。
- 结果出现时强调“稳定落地”，不要像中奖。
- 竞争力分数开启后，可使用小奖杯、轻量彩带、徽章闪光。
- 不使用全屏烟花或长时间循环动画。

## 四、信息架构

V1 页面清单：

1. 首页
2. 城镇职工快速测算页
3. 当前缴费水平选择页 / 半屏弹层
4. 测算结果页
5. 补充个人账户余额页
6. 数据来源 / 关于 / 免责声明页
7. 分享卡片预览页 / 弹层

长期模块在首页展示，但只有城镇职工养老金测算可点击。其他模块置灰，显示“敬请期待”。

## 五、首页设计

首页结构：

```text
顶部：退休金算盘
副标题：少填也能算，细填更准确

主入口：
城镇职工养老金测算
约 30 秒完成快速估算

辅助标签：
不登录 / 不上传 / 可补充变准

规划中模块：
机关事业养老金测算        敬请期待
城乡居民养老金测算        敬请期待
企业年金测算              敬请期待
职业年金测算              敬请期待
个人养老金测算            敬请期待

底部：
结果为估算参考，最终以社保经办机构核定为准
```

交互：

- “城镇职工养老金测算”可点击。
- 其他模块置灰，不进入详情页。
- 点击置灰模块时，可轻提示：“该模块正在规划中，敬请期待。”
- 置灰模块不可表现为主 CTA，不使用高亮色按钮。
- 置灰模块文案统一为“敬请期待”，不写“即将测算”“可预约”等暗示马上可用的表达。

## 六、快速测算页

页面目标：30 秒内完成填写。

字段顺序：

1. 出生年月
2. 性别 / 退休类型
3. 预计退休 / 待遇领取城市
4. 已缴费年限
5. 当前缴费水平

性别 / 退休类型文案：

- 男性
- 女性，约 55 岁退休
- 女性，约 50 岁退休
- 不确定，帮我估算

不确定处理：

- “不确定，帮我估算”主要用于用户不确定自己应按女性约 50 岁还是约 55 岁规则退休的情况。
- 原型和开发不输出单一确定退休年龄。
- 系统应按两组退休类型分别估算，并在结果页合并为更宽的估算区间。
- 结果页提示：“退休类型可能影响退休时间和养老金金额，当前已按可能规则给出区间估算。”
- 如果用户后续明确选择 50 岁或 55 岁规则，则重新测算并收窄区间。

城市选择：

- 城市为必填项。
- 上海、北京、深圳都展示。
- 每个城市展示数据状态。
- 未达到上线门槛的城市可以展示但不可提交普通测算。

城市状态与提交规则：

| 城市状态 | UI 表现 | 是否可提交 | 点击 / 提交提示 |
| --- | --- | --- | --- |
| 可测算 | 正常文字 + 绿色状态标签 | 是 | 无 |
| 数据复核中 | 可选择查看说明，状态标签为提醒色 | 否 | 该城市数据仍在复核中，为避免误导，暂不开放普通测算 |
| 历史参数补齐中 | 可选择查看说明，状态标签为提醒色 | 否 | 该城市历史参数仍在补齐中，为避免误导，暂不开放普通测算 |
| 未开放 | 置灰展示 | 否 | 该城市暂未开放测算 |

如果三座城市均不可提交：

- 底部主按钮改为“暂未开放测算”。
- 按钮不可用。
- 页面提供“查看数据来源”和“了解上线进度”入口。

城市状态示例：

```text
上海
数据复核中

北京
参数复核中

深圳
历史参数补齐中
```

如果某城市达到上线门槛：

```text
上海
可测算
```

未达门槛城市点击或提交时提示：

```text
该城市数据尚未达到上线门槛，为避免误导，暂不开放普通测算。
```

底部按钮：

- 主按钮：立即测算
- 次级入口：我知道个人账户余额，先补充
- 只有已选择“可测算”城市且必填项完整时，主按钮才可点击。

## 七、当前缴费水平选择

使用大选项卡或半屏弹层，不让用户理解“缴费指数”。

选项：

1. 不清楚，按当地最低档估算
   - 适合不知道社保缴费基数的情况，结果范围会更宽。
2. 按税前月薪估算
   - 系统会按当地缴费上下限处理。
3. 我知道社保缴费基数
   - 结果会比按工资估算更接近。
4. 按社平工资档位估算
   - 60% / 100% / 150% / 300%

规则：

- 可以推荐“不清楚，按最低档估算”。
- 不做默认勾选。
- 必须用户主动确认。

## 八、结果页设计

结果页第一屏优先展示养老金估算结果，而不是竞争力或奖杯。

结构：

```text
你的退休金算盘打出来了

预计每月养老金
约 5200 元/月

估算范围
约 4400 - 6000 元/月

拆分
基础养老金
个人账户养老金
过渡性养老金：暂未纳入
```

随后展示：

- 估算状态标签
- 补充个人账户余额入口
- 关键假设
- 数据质量
- 特殊情况提示
- 免责声明
- 分享按钮

### 估算状态标签

V1 不做百分比准确度条，不展示“准确度 82%”这类伪精确数值。

使用文字标签：

```text
估算状态：快速估算
原因：个人账户余额暂由系统估算，部分历史参数仍在复核
建议：补充个人账户余额后结果会更接近实际
```

用户补充个人账户余额后：

```text
估算状态：账户余额已补充
已使用你填写的个人账户余额
仍未纳入过渡性养老金
```

状态标签枚举：

| 状态标签 | 触发条件 | 文案边界 |
| --- | --- | --- |
| 快速估算 | 用户未填写个人账户余额，系统按规则估算账户余额 | 不写“准确率”，提示结果范围较宽 |
| 账户余额已补充 | 用户填写个人账户余额并用于重算 | 不写“较准”或“精确”，只说明已使用用户填写值 |
| 参数复核中 | 当前城市或年份参数未完成复核，但页面用于内部原型或灰度 | 不开放普通测算，或必须明显提示不可作为正式结果 |
| 特殊情况未纳入 | 用户可能存在视同缴费、多地缴费、提前退休等情况 | 不估特殊部分金额，只提示会影响实际结果 |

### 竞争力模块

竞争力模块可以做，但必须受 benchmark feature flag 控制。默认状态为隐藏，不做占位卡。

未满足闸门时：

- 不展示竞争力分数。
- 不展示百分位。
- 不展示奖杯。
- 不展示轻量彩带。
- 不展示竞争力分享卡片。

满足闸门后，在结果页第二屏展示：

```text
退休金竞争力
78 分
预计超过 78% 的上海同城估算样本
```

补充说明：

```text
该结果基于固定估算样本，不代表真实用户排名。
```

视觉：

- 可以使用小奖杯或徽章。
- 高分可以使用轻量彩带动效。
- 不使用全屏烟花。
- 不让竞争力模块盖过养老金估算结果。

开启闸门条件：

- `benchmarkVersion` 非空。
- benchmark 样本文件存在。
- benchmark 版本已固定。
- 当前城市参数版本已复核。
- 10-20 个金额样例通过校验。
- 反直觉校验通过。
- 文案明确为“同城估算样本”，不写真实用户排名。

## 九、个人账户余额补充页

页面结构：

```text
标题：补充个人账户余额
说明：如果你能查到个人账户余额，结果会更接近实际；不知道也可以跳过

输入项：
当前个人账户余额     元

主按钮：重新测算
次按钮：暂时不知道，继续使用系统估算
```

填写后回到结果页，并标记：

```text
个人账户余额已使用你填写的数值
```

## 十、数据来源 / 关于页

展示内容：

- 数据来源
- 参数版本
- 模型版本
- 隐私说明
- 免责声明

入口位置：

- 首页底部免责声明文字可点击进入。
- 结果页“数据质量”区可点击进入。
- 城市不可算提示中提供“查看数据来源”入口。
- 关于页内展示参数版本、模型版本和最近更新时间。

推荐文案：

```text
本工具使用公开政策参数和本地模型估算，不是官方待遇核定。
```

```text
你的测算信息仅保存在本机，不会上传服务器。
```

```text
部分城市或年份参数未达到上线门槛时，不开放普通测算。
```

## 十一、分享卡片

默认分享卡片不展示金额。

基础 V1 文案：

```text
退休金算盘
我用退休金算盘估了一下退休金
少填也能算，细填更准确
扫码也来算算
```

竞争力模块满足闸门后，可增加分享卡片版本：

```text
退休金算盘
我的退休金竞争力：78 分
预计超过 78% 的上海同城估算样本
少填也能算，细填更准确
```

分享仍默认不展示：

- 具体养老金金额
- 出生年月
- 缴费年限
- 个人账户余额
- 真实用户排名

V1 原型不展示“分享时展示金额”开关。该能力仅作为后续用户主动授权后的增强项，不进入当前原型验收范围。

## 十二、V1 不出现

V1 不出现：

- 排行榜
- 真实用户排名
- “打败全国 X% 用户”
- “官方认证”
- “精确测算”
- “保证领取”
- 默认分享养老金金额
- “分享时展示金额”开关
- 登录、手机号、身份证、真实姓名
- 历史缴费分段长表单
- 多地缴费精算
- 过渡性养老金金额

V1 可出现但有闸门：

- 竞争力分数
- 同城估算样本百分位
- 小奖杯 / 徽章
- 轻量彩带动效
- 竞争力分享卡片

闸门条件：

- benchmark 样本已生成。
- benchmark 版本已固定。
- `benchmarkVersion` 非空。
- 样本文件存在。
- 当前城市参数版本已复核。
- 10-20 个金额样例通过校验。
- 分数通过反直觉校验。
- 文案明确为“同城估算样本”，不写真实用户排名。
