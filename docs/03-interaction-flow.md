# 03 系统交互与页面流程

更新时间：2026-05-28

## 总体交互原则

退休金算盘的用户年龄可能偏大，页面交互必须尽可能简单。

设计原则：

- 不用复杂术语开场。
- 不让用户一次填完所有字段。
- 支持“不知道”“大概”“按最低档估算”。
- 重要字段用选项代替解释。
- 复杂字段折叠到“提高准确度”里。
- 结果页给出下一步建议，而不是只给结果。

## 首页流程

首页展示产品名称、核心说明和模块入口。

建议结构：

```text
顶部：退休金算盘
副标题：少填也能算，细填更准确
主入口：城镇职工养老金测算
其他模块：机关事业 / 城乡居民 / 企业年金 / 职业年金 / 个人养老金，置灰，敬请期待
底部：测算结果仅供参考，最终以社保经办机构核定为准
```

首页模块状态：

- 城镇职工：可点击进入。
- 其他模块：置灰展示，显示“敬请期待”，点击后只做轻提示。

## 快速测算流程

快速测算页目标是让用户在 30 秒内完成输入。

页面结构：

```text
标题：先算个大概
说明：只填 5 项，马上估算退休后每月能领多少

1. 出生年月
2. 性别 / 退休类型
3. 退休城市
4. 已缴费年限
5. 当前缴费水平

按钮：立即测算
次级入口：我知道个人账户余额
```

性别 / 退休类型选项：

```text
男性
女性，约 55 岁退休
女性，约 50 岁退休
不确定，帮我估算
```

MVP 不在快速测算页询问：

- 是否多地缴费。
- 是否特殊工种。
- 是否病退。
- 是否有断缴补缴。
- 是否有视同缴费。
- 是否有企业年金或职业年金。

字段体验：

- 出生年月用年月选择器。
- 已缴费年限可以用数字输入，也可以用滑块。
- 当前缴费水平用卡片选项，避免用户理解“缴费指数”。
- 退休城市为必填项，展示上海、北京、深圳及数据状态。
- 用户不清楚时，推荐按当地最低档估算，但需要用户主动确认。

## 当前缴费水平选择流程

用户点击“当前缴费水平”后，展示四个选项：

```text
不清楚，按当地最低档估算
适合不知道社保缴费基数的用户

按税前月薪估算
适合知道自己每月工资的用户

我知道社保缴费基数
适合能查到社保缴费基数的用户

按社平工资档位估算
60% / 100% / 150% / 300%
```

系统处理：

- 月薪和缴费基数都要按当地缴费基数上下限进行裁剪。
- 档位选择转换为缴费指数。
- 不知道时使用当地最低缴费基数，结果标记为低准确度。

## 结果页流程

结果页是建立信任和引导补充信息的关键页面。

建议结构：

```text
主结果：预计每月养老金
区间：预计范围
拆分：基础养老金、个人账户养老金、过渡性养老金暂未纳入
关键假设：按当前缴费水平持续到退休
数据质量：当前参数完整度和个人账户余额状态
估算状态：当前为快速估算，建议补充个人账户余额
按钮：提高准确度
按钮：分享给朋友
```

结果页提示文案：

```text
这是根据当前填写信息得到的估算结果，实际养老金以社保经办机构核定为准。
```

补充信息引导：

```text
补充个人账户余额后，可以减少个人账户养老金估算误差。
```

## 补充信息流程

V1 补充信息页只处理个人账户余额，不做长表单。

页面结构：

```text
标题：补充个人账户余额
说明：如果你能查到个人账户余额，结果会更接近实际情况；不知道也可以跳过

输入项：
当前个人账户余额

底部按钮：重新测算
```

状态：

- 未填写
- 已填写
- 跳过，按系统估算

已填写时：

- 展示摘要。
- 提供“修改”入口。
- 结果页标记“个人账户余额已使用用户填写值”。

## 分享流程

分享触发点：

- 结果页主按钮。
- 结果页底部分享卡片。

默认分享图内容：

```text
退休金算盘
我用退休金算盘估了一下退休金
少填也能算，细填更准确
扫码也来算算
```

默认不展示：

- 用户具体养老金金额。
- 出生年月。
- 缴费年限。
- 个人账户余额。
- 竞争力分数和百分位，除非 benchmark 样本已通过校验。

后续可选：

- “分享时展示预计金额”开关，后续再做，不进入 V1 原型验收范围。
- 竞争力分数分享卡片，必须受 benchmark 闸门控制。

## 视觉效果

结果页可以有轻微仪式感，但基础结果第一屏不以奖杯、烟花、彩带作为第一视觉。养老金测算首先要让用户相信结果来源和边界。

benchmark 通过后可在竞争力模块引入分数等级视觉：

| 分数 | 视觉 | 文案方向 |
| --- | --- | --- |
| 85-99 | 小奖杯 / 徽章 + 局部轻量动效 | 退休金竞争力很强 |
| 70-84 | 徽章 + 轻量彩带 | 表现不错 |
| 50-69 | 徽章 + 轻微闪光 | 处于中等水平 |
| 1-49 | 温和鼓励 | 可以通过补充信息获得更准结果 |

## 异常与边界状态

应支持：

- 缴费年限不足最低年限时，提示可能无法按月领取或需要延缴。
- 出生年月导致退休年份太远时，提示政策可能变化。
- 用户未填写个人账户余额时，用估算值，并明确标注。
- 用户填写明显异常数据时，做温和提示，不直接阻断。

MVP 暂不主动询问的复杂情况：

- 多地缴费。
- 特殊工种提前退休。
- 病退、退职。
- 视同缴费和过渡性养老金。
- 断缴、补缴、一次性缴费。

这些情况统一在结果页免责声明中提示，不进入第一版计算流程。
