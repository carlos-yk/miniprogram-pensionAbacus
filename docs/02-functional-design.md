# 02 功能设计

更新时间：2026-05-28

## 首页模块规划

首页可以规划为 6 个模块，形成长期产品结构。

1. 城镇职工养老金测算
2. 机关事业养老金测算
3. 城乡居民养老金测算
4. 企业年金测算
5. 职业年金测算
6. 个人养老金测算

V1 只实现第一个模块：城镇职工养老金测算。

V1 首页展示 6 个模块，其中只有“城镇职工养老金测算”可点击。其他模块置灰展示，并标记“敬请期待”，避免用户误解产品已经完整支持。

## V1 MVP 范围

第一版目标是先解决绝大部分正常城镇职工用户的快速估算，不追求覆盖所有特殊政策场景。

MVP 包含：

- 单城市测算：上海、北京、深圳。
- 正常城镇职工基本养老保险估算。
- 正常法定退休年龄推算。
- 当前缴费水平持续到退休的快速估算。
- 个人账户余额未知时系统估算，已知时优先使用用户填写值。
- 缴费年限不足最低年限时给出提醒。
- 结果页展示总额、区间、基础养老金、个人账户养老金、关键假设和免责声明。

MVP 不精算：

- 多地缴费待遇领取地判断。
- 特殊工种提前退休。
- 病退、退职、个案审批。
- 机关事业单位养老。
- 城乡居民养老。
- 企业年金、职业年金、个人养老金。
- 过渡性养老金金额。
- 断缴、补缴、一次性缴费等复杂历史。
- 排行榜。
- 未通过 benchmark 校验的竞争力分数。

处理原则：

- 用户遇到特殊情况时，结果页只提示“该情况可能影响结果，当前版本暂未纳入精算”。
- 不因为特殊场景阻断普通用户测算。
- 不在第一版引导用户填写大量复杂历史信息。

## V1 核心模块

模块名称：城镇职工养老金测算

支持城市：

- 上海
- 北京
- 深圳

核心流程：

```text
进入模块 -> 快速填写基础信息 -> 立即测算 -> 查看结果 -> 补充个人账户余额 -> 重新测算 -> 分享
```

## 快速测算必填字段

V1 快速测算只要求最小必要字段。

| 字段 | 用途 | 输入方式 | 备注 |
| --- | --- | --- | --- |
| 出生年月 | 推算退休年龄和预计退休年份 | 年月选择器 | 允许只选年月 |
| 性别 / 退休类型 | 判断退休年龄规则 | 单选 | 男性、女性约 55 岁退休、女性约 50 岁退休、不确定 |
| 预计退休城市 | 选择城市参数 | 单选，必填 | 上海、北京、深圳，展示数据状态 |
| 已缴费年限 | 基础养老金计算 | 数字输入或滑块 | 允许填大概年限 |
| 当前缴费水平 | 推算缴费指数和未来缴费 | 多选项入口 | 见下一节 |

## 当前缴费水平输入方式

用户不一定知道缴费基数，所以需要提供多种入口。

推荐选项：

1. 不清楚，按当地最低档估算
2. 按税前月薪估算
3. 我知道社保缴费基数
4. 按社平工资档位估算：60% / 100% / 150% / 300%

推荐规则：

- 对普通用户，推荐“不清楚，按当地最低档估算”，但需要用户主动确认，不做无感默认。
- 如果用户填了税前月薪，系统按当地缴费基数上下限裁剪。
- 如果用户知道缴费基数，优先使用缴费基数。
- 如果用户选档位，则用当地社平工资或缴费基准推算。

## 补充信息模块

用户完成快速测算后，可以点击“提高准确度”继续补充信息。

推荐入口文案：

- 提高准确度
- 补充信息重算
- 算得不准？继续补充
- 继续补充，让结果更准

已确认补充模块：

| 模块 | V1 状态 | 作用 |
| --- | --- | --- |
| 个人账户余额 | 重点实现 | 直接影响个人账户养老金 |
| 历史缴费记录 | V1.1 / V2 | 提升平均缴费指数准确度，V1 不做复杂分段录入 |
| 多地缴费经历 | 结果页提示 | V1 仅提示精度降低，后续支持待遇领取地判断 |
| 未来缴费计划 | V1 默认假设 | V1 默认按当前缴费水平持续到退休，复杂配置后续做 |
| 早年工作 / 视同缴费 | 仅提示 | V1 不估金额，只提示过渡性养老金可能未纳入 |
| 断缴补缴经历 | 后续 | V1 不精算 |
| 特殊退休情况 | 后续 | 如提前退休、特殊工种等，V1 不做完整判断 |

## 补充模块交互规则

- V1 只提供个人账户余额补充入口。
- 用户不知道个人账户余额时可以跳过，结果页必须标注该项为估算。
- 补充完成后返回结果页，自动重新计算。
- 结果页展示“当前数据质量”和“个人账户余额是否已填写”。
- V1 使用“估算状态标签”，不展示百分比准确度条。
- 历史缴费、多地缴费、视同缴费等复杂因素只作为结果页风险提示或后续入口，不影响 V1 主流程。

## 结果页功能

结果页不要只展示一个总数，而要拆分展示。

建议展示：

- 预计每月养老金总额
- 估算区间，例如“预计 4400 - 6000 元/月”
- 基础养老金
- 个人账户养老金
- 过渡性养老金未纳入提示
- 关键假设，例如“未来按当前缴费水平持续到退休”
- 数据质量提示
- 估算状态标签
- 影响结果的关键因素
- 个人账户余额补充入口
- 分享入口

可选展示：

- 退休金竞争力分数
- 超过同城估算样本百分比
- 小奖杯 / 徽章
- 轻量彩带动效

以上两项只有在 benchmark 样本生成、通过样例校验并版本化后才展示。

过渡性养老金 V1 处理：

- V1 不要求用户填写早年工作/视同缴费信息。
- V1 不展示过渡性养老金金额。
- 结果页统一提示“如存在视同缴费或早年工作经历，实际养老金可能与估算不同，最终以当地社保核定为准”。

## 分享功能

分享目标：

- 引导用户把小程序分享给朋友，让朋友参与测算。
- 结果页要有适合截图/分享的视觉效果。

默认分享内容：

- 展示产品名称和普通测算入口文案。
- 默认不展示具体养老金金额，保护隐私。

benchmark 通过后可升级为：

- 展示退休金竞争力分数。
- 展示“预计超过 X% 的同城估算样本”。

可选后续能力：

- 用户主动选择“展示金额”后，分享图才展示预计金额，但不进入 V1 原型验收范围。
- 分享图支持城市、分数、称号、奖杯等级，但必须受 benchmark 闸门控制。

## 本地缓存

V1 不登录、不保存云端数据，但允许本地缓存。

缓存内容：

- 最近一次测算输入。
- 最近一次结果。
- 个人账户余额是否已补充。

隐私说明建议：

```text
你的测算信息仅保存在本机，不会上传服务器。
```
