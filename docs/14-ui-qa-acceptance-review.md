# 14 UI 设计 QA 验收报告

更新时间：2026-05-28

本文档记录 QA agent 对小程序 UI 设计方案的验收结论、风险项和进入低保真 / 中保真原型前的验收标准。

## 一、验收结论

结论：有条件通过。

当前 UI 方向可以进入低保真 / 中保真原型，但原型必须明确表达：

- 哪些城市可测算，哪些城市不可提交。
- 哪些能力被数据闸门隐藏。
- 哪些结果是估算，不是官方核定。
- 哪些分享内容默认不展示，避免泄露个人金额。

当前不建议直接进入高保真定稿或开发冻结。

## 二、P0 阻断问题

无。

## 三、P1 高优先级问题

### 1. 城市数据门槛和不可算状态必须硬闭环

问题：

城市必填和不可算方向正确，但如果原型中上海、北京、深圳看起来都能直接提交，会误导用户。

处理要求：

- 只有“可测算”城市允许提交。
- “数据复核中”“历史参数补齐中”“未开放”城市不可提交。
- 若三城均不可提交，主按钮改为“暂未开放测算”。
- 城市不可算提示提供“查看数据来源”入口。

已落文档：

- [13 UI 风格与交互设计](./13-ui-style-and-interaction-design.md) 已新增“城市状态与提交规则”。

### 2. “不确定，帮我估算”不能输出伪确定结果

问题：

女性约 50 岁 / 55 岁退休类型会影响退休年份、计发月数和未来缴费年限。用户选择“不确定”时，不能给出单一确定退休年龄。

处理要求：

- 按女性约 50 岁和约 55 岁两组规则分别估算。
- 结果页合并展示更宽的估算区间。
- 提示“退休类型可能影响退休时间和养老金金额”。
- 用户后续明确退休类型后，再重新测算并收窄区间。

已落文档：

- [13 UI 风格与交互设计](./13-ui-style-and-interaction-design.md) 已新增“不确定处理”。

### 3. 竞争力模块默认隐藏，必须受 benchmark feature flag 控制

问题：

当前 benchmark 尚未生成。若原型默认展示竞争力分数、百分位、奖杯、彩带或竞争力分享卡，会突破数据门槛。

处理要求：

- benchmark 未满足闸门时，竞争力模块完全隐藏，不做占位卡。
- 不展示竞争力分数。
- 不展示百分位。
- 不展示奖杯、彩带。
- 不展示竞争力分享卡。

开启闸门条件：

- `benchmarkVersion` 非空。
- benchmark 样本文件存在。
- benchmark 版本已固定。
- 当前城市参数版本已复核。
- 10-20 个金额样例通过校验。
- 反直觉校验通过。
- 文案明确为“同城估算样本”，不写真实用户排名。

已落文档：

- [13 UI 风格与交互设计](./13-ui-style-and-interaction-design.md) 已补齐 feature flag 规则。

### 4. “估算状态标签”需要枚举，避免“较准”这类过度承诺

问题：

“较准”容易被理解为有准确率算法支持。即使填写个人账户余额，过渡性养老金、特殊退休、多地缴费等仍可能影响实际结果。

处理要求：

- 不使用“较准”“低准确度”“准确率 X%”。
- 使用可解释的状态标签。

推荐枚举：

- 快速估算
- 账户余额已补充
- 参数复核中
- 特殊情况未纳入

已落文档：

- [13 UI 风格与交互设计](./13-ui-style-and-interaction-design.md) 已新增“状态标签枚举”。

## 四、P2 中优先级问题

### 1. 首页置灰模块需要视觉验收标准

处理要求：

- 置灰模块不可表现为主 CTA。
- 点击只 toast，不进入详情页。
- 文案统一“敬请期待”。
- 不使用“即将测算”“可预约”等暗示马上可用的文案。

### 2. 分享金额开关不进入 V1 原型

处理要求：

- V1 原型只保留普通分享卡。
- 默认不展示养老金金额。
- 不展示“分享时展示金额”开关。
- 竞争力分享卡必须受 benchmark 闸门控制。

### 3. 移动端适老状态需要进入原型验收

处理要求：

- 覆盖 375px 宽屏。
- 覆盖较大字号。
- 底部固定按钮避开安全区。
- 键盘弹起时输入框不被遮挡。
- 表单错误提示显示在字段附近。
- 结果页长文案自然换行，不遮挡后续内容。

### 4. 动效边界需要克制

处理要求：

- 只允许局部、一次性、短时动效。
- 不循环闪烁。
- 不做全屏烟花。
- 不影响结果阅读。

### 5. 数据来源 / 免责声明入口需要可达

处理要求：

- 首页底部可进入。
- 结果页数据质量区可进入。
- 城市不可算提示可进入。

## 五、进入原型前验收标准

低保真 / 中保真原型必须覆盖：

- 首页 6 模块状态：城镇职工可点击，其余置灰 toast。
- 城市状态表：可测算、复核中、补齐中、未开放。
- 三城均不可算时的按钮和替代出口。
- “不确定，帮我估算”的双规则区间展示。
- 估算状态标签枚举和触发条件。
- benchmark 未生成时竞争力模块完全隐藏。
- 默认分享卡不含金额、出生年月、缴费年限、账户余额。
- 城市不可算、缴费年限不足、账户余额跳过、参数复核中的异常态。
- 375px 移动端、较大字体、底部安全区、长文案换行。

## 六、当前处理状态

已处理：

- 城市状态与提交规则补入 UI 文档。
- 退休类型不确定处理补入 UI 文档。
- 估算状态标签改为枚举。
- “较准”改为“账户余额已补充”。
- benchmark feature flag 条件补齐。
- 分享金额开关明确不进入 V1 原型。
- 首页置灰模块视觉边界补齐。
- 数据来源入口位置补齐。

仍需后续设计原型验证：

- 具体页面线框。
- 关键异常态线框。
- 移动端安全区和键盘态。
- 分享卡实际视觉。
- 数据来源 / 免责声明页实际入口位置。
